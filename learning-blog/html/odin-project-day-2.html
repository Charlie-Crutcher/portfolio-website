Lesson 1 - Working with Text

Most content on the web is text-based, which means working with HTML text elements is a common occurance when building web pages.


Paragraphs.

If we wanted to create paragraphs in HTML, we need to use the paragraph element, which will add a new line after each paragraph. A paragraph element is defined by wrapping text content with a <p> tag.

Headings.

Headings are different from other HTML text elements, as they are displayed larger and bolder than other text to distinguish they are headings.

There are six levels of headings, starting from <h1> being the largest, and <h6> being the smallest.

Using the correct level of heading is important, as levels provide a hierarchy to the content. A h1 heading should always be used for the heading of the overall page, with lower level headings being used for headings in small sections of the page.

Strong Element.

The strong element makes text bold. It also semantically marks text as important, which affects accessibility tools such as screen readers. The tone of voice on some screen readers will change to communicate the importance of the text within a strong element. To define a strong element, we simply wrap text in a <strong> tag.

Em Element.

The em element makes text italic. It also semantically places emphasis on the text.

Nesting and Indentation.

Any time you indent elements within another element, this is called nesting elements.

This creates a parent and a child relationship between the two elements. The nested elements will be the children of the parent it is nested within.

Nesting and indentation makes our HTML much easier to read, both for ourselves and for any possible future developers.

The parent, child and sibling relationships between elements becomes much more important when adding CSS and JavaScript, but for now it is important to know the distinction between how elements are related, and the terminology used to describe their relationships.

HTML Comments.

HTML comments are not visible to the browser, allowing us to comment on our code so that other developers or our future selves can read them to get some context about the code written.

Comments in HTML can be written with the <!-- and --> tags.





Lesson 2 - Lists

Lists are everywhere on the web, and you are eventually going to need to use them. Luckily though, HTMl offers a couple of different types of lists we can implement.


Unordered Lists.

If you want to have a list of items where the order isn't particularly important, such as a shopping list of items, then you can use an unordered list.

Unordered lists are created using the <ul> element, with each item within the list being nested within this ul element with the <li> element.

Ordered Lists.

When you need to create a list of items where the order does matter, such as a top 10 list, then you can use an ordered list.

Ordered lists are created using the <ol> element, with each item within the list being declared the same as the unordered list. The only difference is that these items will be numbered, from top to bottom.

Unordered List Example.

Unordered Shopping List:

  • Milk
  • Bread
  • Butter
  • Cereal
  • Cheese

Ordered List Example.

Top Five Films:

  1. Taxi Driver
  2. Defiance
  3. Lord of the Rings
  4. Troy
  5. A Bridge too Far.




Lesson 3 - Links and Images.

Links are one of the key features of HTML. Links allow us to link to other HTML pages on the web, which is why it's called the web!

Anchor Elements.

To create a link in HTML, we use the anchor element. An anchor element is defined by wrapping text or elements we want to become a link in a <a> tag.

Just an anchor element will not link us to any web page however, as we need to tell it where to take us. We can do this by using a HTML attribute.

A HTML attribute or href, gives additional information to an element and always goes in the element's opening tag. An attribute is usually made up of two parts: a name, and a value, but not all attributes require a value.

To make an anchor tag redirect us to a different web page, we can combine the anchor tag with a HTML attribute.

Opening Links in a New Tab.

In order to open an anchor element's link in a new tab, we need to add another attribute: the target attribute.

While href specifies the destination link, target specifies where the linked resource will be opened. If it is not present, by default, it will take on the _self value, which opens the link in the current tab. To open the link in a new tab, you can set the target attribute to _blank.

Relationship Attribue.

The relationship, or rel attribute is used to describe the relation between the current page and the linked document in an anchor tag.

Example: rel="noopener noreferrer"

The noopener value prevents the opened link from gaining access to the webpage from which it was opened.

The noreferrer value prevents the opened link from knowing which webpage or resource has a link (or reference) to it.

Why add these? Security. The prevention of access that is caused by the noopener prevents phishing attacks.

Absolute and Relative Links.

There are generally two kinds of links used with HTML; links to pages on other websites, and links to pages located on our own website.

Absolute Links are links to pages on other websites on the internet. The entire address of the website must be used, such as https://www.example.com

Relative Links are links to other pages within our own website. These do not include the domain name, as it is simply another page on our site.

If our relative link redirects us to a page in a different directory, we must reference this directory in the link, such as pages/example.html

Images.

Websites often contain images, as without them they may seem fairly boring. Luckily however, HTML provides a wide variety of elements for displaying different types of media, with the most widely used of these being the image element.

To display an image in HTML we use the <img> element. Unlike the other elements we have used, the img element is a void element, meaning it does not need a closing tag.

Instead of wrapping content with an opening and closing tag, it embed an image into the page using a src attribute, telling the browser where the image file is located. The src attribute works much like the href attribute for anchor tags. Images can also be embedded using both absolute and relative paths, also seen in the achor element.

Alt Attribute.

Besides the src attribute, every image element must also have an alt (alternative text) attribute.

This is used to describe an image, which will be in the image's place if it cannot be loaded. It is also used with screen readers to describe the image to visually impaired users.

Image Size Attributes.

While not required, specifying the height and width of an image helps the browser layout the page without causing the page to jump and flash.

It can be good practice to always specify these attributes on every image, even when the image is the correct size, or you are using CSS to modify it.

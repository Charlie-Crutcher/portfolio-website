Lesson 1 - What is HTML, CSS and JavaScript?

What are the main three languages used?

1: What is HTML? It is a language that determines how documents and web pages are displayed in a web browser, the language for the building blocks of any website.

2: What is CSS? CSS, or 'Cascading Style Sheets', is a popular style sheet language that determines how a document in HTML is styled, such as it's colours, fonts, layout and responsive features.

3: What is JavaScript? JavaScript allows you to change CSS and HTML elements on your website after the site has been loaded, giving you the ability to make your site more interactive and engaging for users with animations, embedded video media and more.

These three programming languages typically lay the foundations of any website, which is why it is important that you are aware of how these languages function, how they may affect your website and what they are typically used for.


TL:DR

HTML Creates the structure

CSS Stylizes the website

JavaScript Increases interactivity


Additional Languages

Websites may use a wide range of different programming languages, with some being more complex than others.

Some of the more common web dev languages consist of HTML, CSS, JS, PHP, Python, Ruby and SQL, however, the primary languages (HTML, CSS and JS) should be enough to understand the basics.





Lesson 2 - Elements and Tags

HTML defines the structure and content of webpages. HTML elements are what creates all of the paragraphs, headings, lists, images and links that make up a typical website.

Elements and Tags

Almost all elements on a HTML page are just pieces of content wrapped in opening and closing tags.

Opening tags tell the browser this is the start of a HTML element, and closing tags tell the browser where the element ends. They both consist of angle bracks with a word in them. <p>


HTML has a vast list of predefined tags, and it is important to use the correct tags for content for two reasons.

Reason 1: It has a direct impact on how the website is ranked in search engines.
Reason 2: It increases accessibility for users who rely on assitive technologies such as screen readers.
Using the correct elements for content is called semantic HTML.


Void Elements

Some HTML elements don't have a closing tag, instead just having a single tag such as <br>. These are known as void elements becauase they are void of any content, due to them having no closing tag.

Lesson 3 - HTML Boilerplate

All HTML documents have the same basic structure or boilerplate, that needs to be in place before anything useful can be done.

The DOCTYPE

Every HTML page starts with a ductype declaration. The doctype's purpose is to tell the browser what version of HTML it should use to render the document. The latest version of HTML is HTML5, and the doctype for that version is !DOCTYPE html.

After the doctype is declared, you need to provide a <html> element. This is what's known as the root element of the ducment, meaning that every other element in the document will be a descendant of it.

This becomes more important later on when we learn about manipulating HTML with JavaScript, but for now, just know that the <html> element should be included on every HTML document.

The lang attribute specifies the language of the text content in that element. This attribute is primarily used for improving accessibility of the webpage, allowing for assistive technologies to adapt according to the language.

Head Element

The <head> element is where we put important meta-information about our webpages, and stuff required for our webpages to render correctly in the browser. Inside the head element, we should not use any element that displays content.

Meta Element

A meta tag should always be included in the head element, as it ensure that the webpage will display special symbols and characters from different languages correctly in the browser.

Title Element

The title element is used to change the text in the webpage's browser tab.

Body Element

The final element needed to complete the HTML boilerplate is the body element. This is where all the content that will be displayed to users will go; the text, images, lists etc.

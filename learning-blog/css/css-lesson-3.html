Lesson 3 - Inspecting HTML and CSS

Date: 8th October 2024


Introduction

Being able to inspect and debug HTML and CSS pages is vital for frontend development. This lesson will discuss the Chrome Dev Tools, which allow you to see information about HTML elements and CSS rules, whilst also assisting in finding and fixing problems in the code.


The inspector

The inspector can either be opened by right clicking a webpage and clicking 'inspect', or pressing F12. Don't be overwhelmed by all the tools, as for this lesson we are just focusing on the Elements and Styles panels.


Inspecting elements

Inside the Elements panel, the entire HTML structure of your page can be seen. You can click on any element in this panel to select it.

When an element is selected, the Styles tab will show you all the applied styles, as well as any styles that are being overwritten (these are seen by a strikethrough of the text).


Testing styles in the inspector

The Styles panel allows you to edit styles directly in the browser. You can do this by clicking inside of any individual selector and adding a new rule, or altering an existing rule. By doing this, the webpage will respond to the style changes in real-time, while not affecting the source code in the text editor. This can be extremely useful for quickly testing out various attribute and values without needing to reload the page.

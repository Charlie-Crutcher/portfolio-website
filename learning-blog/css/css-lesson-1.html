Lesson 1 - Introduction to CSS

Date: 7th - 8th October 2024


Introduction

Now we have established how to write the HTML that structures our webpages, we can now start to add some style to our pages, which is what CSS is for.


Basic syntax

At the most basic level, CSS is made up of various rules. These rules are made up of a selector and a semicolon-separated list of declarations, with each of those declarations being made up of a property-value pair.


Selectors

Selectors refer to the HTML elements to which the CSS rules apply to; they are what are being 'selected' for each CSS rule. Below, we will go through the most common subsections used in CSS.

Universal selector

The universal selector selects elements of any time. It consists of a simple asterisk (*).

Type selectors

The type selector (or element selector), selects all elements of the given element type, with the syntax just being the name of the element. For example: we could use div {color: white;}, which would select all div elements.

Class selectors

Class selectors select all elements with the given class, which is just an attribute placed on a HTML element. For example div class="alert" in HTML, would be .alert {color: red;} in CSS, only changing this class.

Class selector syntax within CSS start with a period, followed by the case-sensitive value of the class attribute. Classes aren't required to be specific to a particular element, so you can use the same class on as many elements as you would like.

ID selectors

ID selectors are similar to class selectors. They select an element with the given ID, which again is an attribute placed on a HTML element. The difference between classes and IDs is that an element can only have one ID. It cannot be repeated on a single page and should not contain any whitespace.

ID selector syntax start with a hashtag, followed by the case-sensitive value of the ID attribute. You should use IDs sparingly, as in most instances, simply using classes will suffice.

Chaining selectors

Another way to use selectors is to chain them as a list without any separation.

div class='subsection header'
p class='subsection preview'

Here we have two elements with the subsection class, that have some sort of unique styles. What if we only wanted to apply a separate rule to the element that also has header as a second class? We could chain both the class selectors together in CSS.

.subsection.header {color: red;}

What .subsection.header does is selects any element that has both the subsection and header classes. This can also be used to chain a class and an ID, (.subsection#preview) like so.

Descendant combinator

Combinators allow us to combine multiple selectors differently than either grouping or chaining them, as they show a relationship between the selectors. There are four types of combinators in total, but for now we're only going to discuss the Descendant combinator, which is represented in CSS by a single space between selectors. A descendant combinator will only cause elements that match the last selector to be selected if they also have an ancestor that matches the previous selector.

Something such as .ancestor .child, would select an element with the class 'child' if it has an ancestor with the class ancestor. Another way to think of it is that child will only be selected if it is nested inside ancestor, regardless of how deep that nesting is.

There is no real limit to how many combinators you can add to a rule, so .one .two .three .four would work. It would just select an element that has a class of four if it has an ancestor with the class of three, if that had its own ancestor with a class of two etc.


Properties to get started with

Some CSS properties are going to be used all of the time. Some of the most commonly used properties used in CSS will be discussed below, but this is by no means a complete list. It will however be enough to help get you started.

Color and background-color

The color property sets an element's text color, while background-color sets the background color of an element. It is just annoying how it isn't spelt correctly!

Both of these properties can accept one of several kinds of values. A common value is a keyword, such as a color name like red, or the transparent keyword. They also accept HEX, RGB and HSL values, allowing for further customisation.

Typography basics and text-align

font-family can be a single value or a comma-separated list of values that determine what font an element uses. Each font will fall into one of two categories, a 'font family name', such as "Times New Roman", or a 'generic family name', like serif.

If the user's browser cannot find or doesn't support the first font in a list, it will use the next one, which is why it is best practice to have a generic font family as a fallback, such as "Times New Roman" or serif.

font-size sets the size of the font, whilst font-weight affects the boldness of the text. text-align aligns text horizontally within an element. You can use common keywords such as text-align: center, or text-align: left.

Image height and width

While images aren't the only elements that we are able to adjust the height and width of, we are focusing on them specifically in this case.

By default, an img elements height and width values will be the same as the actual image file's height and width. To adjust the size of the image without causing it to lose its proportions, you could use a value of "auto" for the height property, and simply adjust the width value accordingly.


Adding CSS to HTML

External CSS

This is the most common method of adding CSS to HTML. It consists of creating a separate file for the CSS and linking it to the HTML page, inside the 'head' element. This is done using the <link> element.

The advantages of using external CSS is that:

  1. It keeps our HTML and CSS separate. This keeps the HTML file smaller and cleaner looking.
  2. The CSS can be edited in one place.

Internal CSS

Also known as embedded CSS, this involves adding CSS within the HTML file. With this method, you place all the rules inside a pair of opening and closing <style> tags, which are placed inside of the opening and closing <head> tags in the HTML file. This method does not require a <link> element.

While this method can be useful for adding unique styles to a single page of a website, it doesn't keep things separate, and can cause the HTML file to get pretty big.

Inline CSS

This method allows you to add CSS styles directly into HTML elements, however this method isn't as recommended. This is due to a couple of reasons:

  1. It can quickly become messy, causing bloat to the HTML file.
  2. It can cause unnecessary repetition and more bloat.
  3. Inline CSS overrides the other two methods, which can cause unexpected results.
